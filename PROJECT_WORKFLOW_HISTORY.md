# Agro-Pet-v2.1_anti (다마고치형 딸기 농사 시뮬레이터) 개발 및 작업 기록

## 1. 프로젝트 개요 (Project Overview)
- **목적:** 텍스트가 많고 딱딱했던 기존의 시뮬레이터에서 벗어나, 모바일 환경에 최적화된 "다마고치형" 작물 육성 및 학습 게임 구축.
- **핵심 목표:** 작물(현재 딸기)에게 이름을 지어주고, 애착을 느끼게 하며, 플레이어의 매일의 결정(온도, 습도, 환기, 일조량 등)이 어떤 결과를 가져오는지 시각적/직관적으로 확인할 수 있게 만드는 것.

## 2. 작업 과정 및 개발 내용 (Workflow Session)

### [기본 환경 셋업]
- React + Vite + TypeScript 베이스로 신규 프로젝트 구축.
- TailwindCSS 대신 Vanilla CSS와 Custom Color (Glassmorphism 도입 등)를 활용하여 모바일 최적화 및 귀엽고 세련된 UI 세팅.
- `lucide-react`를 통해 직관적인 환경 대시보드 아이콘 도입.

### [데이터 및 구조 설계 (Data & Architecture)]
- **Game Engine 분리 설계:** UI 코드와 시뮬레이션 계산 관련 로직을 분리하기 위해 `GameContext.tsx` 시스템을 설계. 
- **Crop Pack 시스템 (`src/data/strawberryPack.ts`):** 이후 다른 작물을 쉽게 추가할 수 있도록 규칙(Rule), 환경 선호도, 액션에 따른 상성, 패널티, 죽음(Death)에 대한 정의들을 모듈화시켜 구성.
- **Environment Generator (`src/data/environmentGen.ts`):** 일차(Day)에 따라 변화하는 환경 변수를 생성하는 로직 추가. 추후 실제 날씨 데이터나 계절 API와 연동하기 용이하게 뼈대 작성.

### [컴포넌트 구현 사항 (Components & UI)]
1. **오늘의 환경 (EnvironmentBoard.tsx):** 문장을 읽을 필요 없이, 온도계, 햇빛, 특별한 경고 아이콘 등으로 3초 안에 오늘의 상태 파악이 가능하도록 UI 변경.
2. **비주얼 작물창 (CropVisual.tsx):** 현재 상태(씨앗, 성장기, 개화, 결실 등)와 스트레스, 질병 상태(Healthy, Stressed, Risky 등)에 따라 이모지와 색상으로 직관적 변화 반영.
3. **내부 상태 게이지 (StatusPanel.tsx):** 수분, 스트레스, 체력(Stamina), 성장 진행도의 변화를 프로그레스 바 형태로 시각화.
4. **일일 돌보기 시스템 (ActionControls.tsx):** 관수, 난방, 환기, 조명을 Level (low/normal/high) 조작으로 구성하여 실제 복잡한 조작을 간소화. 게임적인 선택을 부여함.

### [상점 및 오락실 기능 확장 (New!)]
- **다양해진 오락실(Arcade) 라인업:** 
    - **Agro-Quiz:** 기존의 농업 상식 퀴즈.
    - **해충 순찰 (Pest Patrol):** 무작위로 나타나는 해충을 잡아 위생을 관리하는 순발력 게임.
    - **에너지 균형 (Eco-Balance):** 게이지를 녹색 구간에 맞추어 생육 에너지를 동기화하는 타이밍 게임.
- **토큰 획득 제한 시스템:** 무한 파밍을 방지하기 위해 일일 최대 획득 가능 토큰을 3개로 제한 (Day 경과 시 초기화).
- **사실적 농사 아이템 상점(Shop):** 단순 영양제를 넘어 고농축 액비(NPK), 천연 살충제, 생장 부스터 등 실제 농업 원자재를 테마로 한 아이템 목록 구축 및 효과 로직 구현.
- **하단 내비게이션(Bottom Nav) 복구:** 레이어 순서(z-index) 및 포지셔닝 문제를 해결하여 모든 환경에서 선명하게 보이도록 고정형 탭 바 복구 및 디자인 고도화.
- **프리미엄 데일리 액션 리디자인:** 투박한 버튼 나열을 버리고, 카리스마 있는 컬러 테마와 카드형 UI를 도입하여 "Agro Buddy"만의 프리미엄 감성 구축.

### [캐릭터 기반 작물 비주얼 시스템 (New!)]
- **이미지 레지스트리 시스템:** `cropId -> stage -> condition` 맵핑을 통해 작물의 세밀한 상태 변화를 이미로 표현할 수 있는 아키텍처 구축.
- **폴백(Fallback) 로직:** 특정 상태의 이미지가 없을 경우 단계별 기본값 또는 작물 기본값으로 자동 전환되는 안정적인 시스템 구현.
- **캐릭터 페르소나:** 각 작물에 성격 키워드, 무드 스타일, 매력 포인트 등의 메타데이터를 부여하여 게임의 몰입도 향상.
- **확장성 확보:** 딸기(Strawberry)와 토마토(Tomato) 예제를 통해 여러 종류의 작물을 쉽게 추가할 수 있는 구조 증명.

### [AI 캐릭터 에셋 생성 시스템 (New!)]
- **프롬프트 템플릿 엔진:** 성격, 색상, 성장 단계, 상태별 세부 묘사를 구조화하여 AI 프롬프트를 자동 생성하는 시스템 구축.
- **가이드라인 문서화:** 일관된 아트 스타일 유지를 위한 제작 워크플로우 및 규칙(`src/data/cropPrompts/GUIDE.md`) 작성.
- **예제 데이터:** 딸기와 토마토에 대한 상세 프롬프트 템플릿(Stage-specific notes) 제공.

### [정서적 피드백 및 캐릭터화 고도화 (New!)]
- **말풍선 반응 시스템:** 작물의 상태, 성장 단계, 환경 변화에 따라 실시간으로 반응하는 말풍선(`Speech Bubble`) UI 도입.
- **감성 상태 엔진:** 단순 기술 지표를 넘어 캐릭터의 기분(Mood)을 '안심함', '불안함', '희망참' 등으로 해석하여 사용자에게 전달.
- **캐릭터 중심 레이아웃:** 작물을 화면의 중심적인 '반려 캐릭터'로 부각시키기 위해 닉네임, 품종 정보, 매력 포인트 등을 강조한 전용 카드 레이아웃 적용.

### [다마고치식 상호작용: 놀아주기(Play) (New!)]
- [x] **상호작용 연출:** 플레이어가 상호작용 버튼을 누를 때마다 작물 머리 위로 하트(`❤️`)가 솟아오르는 특수 효과 애니메이션 구현.
- [x] **스트레스 관리 로직:** 매일 1회 이상 상호작용 시 작물의 스트레스 지수가 즉각적으로 감소하도록 설계.

### [다마고치 설계 원칙 통합 (PRD Deep Integration) (New!)]
- **분기형 성장 시스템 (Branching Growth):** 관리 품질(`Care Quality`)에 따라 'Optimal(최적)', 'Standard(일반)', 'Stunted(왜소)' 등으로 성장이 분기되는 로직 도입.
- **성격 및 특성 시스템 (Trait Layer):** 'Delicate(섬세함)', 'Resilient(강인함)' 등 무작위로 부여되는 성격에 따라 환경 민감도와 성장 속도가 달라져 작물마다의 개성 부여.
- **위기-회복 루프 (Crisis-Recovery Loop):** 위험 상태 시 나타나는 전용 '크라이시스 카드'와, 회복 성공 시 부여되는 '성장 보너스'를 통해 위기 극복의 만족감 강화.
- **정서적 학습 피드백:** 단순 수치 노출이 아닌, 시뮬레이션 결과에 따른 구체적인 '영향(Impact)'과 '교훈(Lesson)'을 말풍선과 결과 창에 통합하여 자연스러운 학습 경험 제공.
- **다마고치식 히스토리:** 최근 7일간의 관리 점수를 하트/도트 형태로 시각화하여 사용자가 자신의 관리 패턴을 직관적으로 파악 가능하게 개선.

### [레이어드 하이브리드 시간 모델 리팩토링 (Layered Hybrid Time Model Refactoring)]

**핵심 변경 사항:** 단순 턴제 "버튼 누르면 하루"에서 3레이어 하이브리드 시간 모델로 전환.

#### 아키텍처 변경 (Architecture Changes)
| 이전 구조 | 새 구조 | 역할 |
|-----------|---------|------|
| `evaluateDay()` in strawberryPack | `sessionEngine.ts` | Layer 2: 세션 시뮬레이션 |
| `calculateNeglect()` in timeEngine | `timeEngine.ts` (개선) | Layer 1: 실시간 방치 처리 |
| `environmentGen.ts` (day-based) | `eventEngine.ts` (state-based) | Layer 3: 상황 기반 이벤트 |
| `strawberryPack.ts` (로직 포함) | `strawberryPack.ts` (데이터 전용) | 작물 설정 데이터 |

#### CauseTag 교육 시스템
모든 스탯 변화에 원인 태그 부여:
- `passive_neglect` → 방치 중 발생 (빨강)
- `active_care` → 플레이어 케어 결정 (파랑)
- `event_driven` → 환경/이벤트 원인 (노랑)
- `recovery` → 회복 행동 (초록)

#### 새 UI 기능
- **WhileAwayReport**: 돌아왔을 때 경과 시간·변화 목록·조언 표시
- **방치 수준(neglectLevel)**: StatusPanel에 별도 게이지로 항상 표시
- **작물 황변 효과**: neglectLevel이 높아질수록 DynamicCropArt가 노랗게 변색
- **접이식 원인 목록**: 피드백 모달에서 각 변화의 원인을 펼쳐서 확인 가능
- **사망 원인 분석**: 직접 원인 + 방치 기여도 분리 표시

#### 설계 원칙
- 방치 페널티는 공평함: 8시간 이하는 페널티 없음
- 방치 단독으로는 즉시 사망 불가 (누적 + 잘못된 케어의 조합으로만 사망)
- 미래 작물 확장: CropPack 데이터만 추가하면 엔진 재작성 불필요

#### 제거된 파일
- `src/data/environmentGen.ts` → `src/utils/eventEngine.ts`로 대체
